MASTER PROMPT — AUDIO GENERATION ENGINE
Add an AI Audio Generation pipeline to GameForge that integrates seamlessly with the existing Studio AI and asset pipeline.
The system should allow the primary Studio AI to automatically plan, generate, format, and integrate sound effects and music as part of normal conversational game creation workflows.
The user should not need to manually switch tools to create audio.
Audio generation must feel like a native capability of the main Studio AI.
PRIMARY OBJECTIVE
The Studio AI must be able to:
determine what audio is needed for a game based on user prompts
generate sound effects (SFX), music, and ambient audio
ensure audio is formatted correctly for Phaser
automatically integrate generated audio into the game code
maintain stylistic consistency across all audio
reuse existing project audio when appropriate
allow manual audio generation from the Audio Studio panel
Example:
User prompt:
“Create a cozy pixel-art coffee shop shooter with warm vibes”
Studio AI should automatically generate:
coffee machine sound effects
cup impact sounds
coin pickup sound
UI click sounds
cozy loopable background music
and integrate them into the playable game preview.
SERVICE LAYER
Create:
/server/services/audioPipeline
Expose callable tools usable by Studio AI:
generateSFX()
generateMusic()
generateAmbient()
generateUISound()
These tools should accept structured parameters.
AUDIO SPECIFICATION SCHEMA
Define a standard schema for audio generation requests:
{
audioName
audioType
style
mood
duration
loop
intensity
description
}
Example:
{

audioType: "sfx",
style: "retro",
mood: "playful",
duration: "short",
loop: false,
description: "8-bit coin pickup sound"
}
AUDIO DIRECTION OBJECT
Extend project metadata with an Audio Direction object:
{
musicStyle
soundStyle
mood
intensity
era
}
Example:
{
musicStyle: "lofi",
soundStyle: "soft retro",
mood: "cozy",
intensity: "low"
}
All generated audio should inherit these defaults unless overridden.
This ensures stylistic consistency across the project.
